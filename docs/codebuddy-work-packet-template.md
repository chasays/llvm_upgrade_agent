# CodeBuddy Work Packet Template

Use this template when MiniMax M2.5 is only available through CodeBuddy.

```text
Task:
Resolve or diagnose the LLVM upgrade issue described below. Make the smallest safe change.

Patch intent:
<commit message and downstream purpose>

Risk level:
<low | medium | high>

Files allowed to edit:
<explicit paths>

Context:
<base/ours/theirs, build errors, lit output, API grounding, Gerrit/xwiki/trilium summaries>

Rules:
- Preserve downstream metax behavior unless the packet explicitly says it is obsolete.
- Do not invent LLVM APIs. Use only APIs shown in the grounding section or found in the local source tree.
- Do not update tests unless the packet says the test has autogenerated CHECK lines.
- If the change touches IR transforms, TableGen, debug info, ABI, or codegen lowering, mark it for human review.

Expected output:
- A concise explanation of the intended edit.
- The code edit.
- Any verification command the runner should execute next.
```
