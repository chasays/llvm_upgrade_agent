# Lit Failure Triage

Classify lit failures before editing code or regenerating checks.

## Workflow

1. Run lit with `-vv --dump-input=always`, or provide an existing log.
2. Check whether the test has autogenerated CHECK lines.
3. Classify each failure:
   - `PURE_TEXT_DRIFT`: likely update-test-checks candidate.
   - `DOWNSTREAM_BREAK`: metax-only target/feature or private behavior involved.
   - `SEMANTIC_REGRESSION`: crash, assertion, wrong IR/asm semantics, verifier failure.
   - `INFRA_OR_FLAKE`: timeout, missing binary, path/environment issue.
4. Call `update-test-checks` only for autogenerated pure text drift.

## Command

```bash
python3 skills/lit-failure-triage/scripts/triage_lit_failure.py \
  llvm/test/CodeGen/Metax/foo.ll \
  --llvm-build /path/to/build
```

From an existing log:

```bash
python3 skills/lit-failure-triage/scripts/triage_lit_failure.py TEST --from-log /tmp/lit.log
```
